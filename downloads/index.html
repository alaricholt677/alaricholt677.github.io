Controls

Movement

WASD to move, mouse drag to look, number keys 1–9 to select hotbar slots.

Interaction

B breaks, P places, right-click interacts, T opens chat, ESC pauses, F5 saves, F6 toggles follower, F7 switches dimension.

F7 Safety

F7 should use a held-key guard so holding it does not spam dimension swaps.

Chat System

Chat is opened with T. Built-in commands are part of ChatOverlay. Mod commands are read from loaded packs.

/help — Show built-in commands and loaded modpack commands
/mods — Show modpack commands registered by loaded packs
/pos — Show position
/tp x y z — Teleport
/give id — Give block/item ID
/dim or /dimension — Switch dimension
/time 6000 or /time set noon — Set time
/summon baby | human | holographic human | boss — Built-in summon commands
/weather clear|rain|snow|thunder|random — Change weather
Pack commands only appear after that pack registers them.

IRL Simulator Extension Pack Creation Guide
==========================================

This README explains how to make an IRL Simulator extension pack / mod pack.

IRL Simulator expansion packs are ZIP files placed in:

expansion_packs/

Each pack contains a required:

pack_manifest.json

and optional JSON files for blocks, entities, commands, recipes, menus, block actions, entity actions, custom AI, and future data.

The mod system is data-driven. Packs do not need Java files. You create JSON data, zip it, and the engine loads it.

BASIC PACK STRUCTURE
--------------------

My_Extension_Pack/
  pack_manifest.json
  README.txt
  content/
    blocks.json
    entities.json
    commands.json
    recipes.json
    menus.json
    block_actions.json
    entity_actions.json
    custom_ai.json
    robots.json
    lights.json
    science.json
  data/
    blocks.json
    entities.json
    commands.json
    recipes.json
    menus.json
    block_actions.json
    entity_actions.json
    custom_ai.json
    robots.json
    lights.json
    science.json
    info.json
    pack_id.txt

IMPORTANT ZIP RULE
------------------

Correct:
my_pack.zip
  pack_manifest.json
  content/blocks.json

Wrong:
my_pack.zip
  my_pack/pack_manifest.json

pack_manifest.json must be at the root of the ZIP.

SUPPORTED SYSTEMS
-----------------

blocks
commands
entities
custom cube models
recipes
menus
block right-click actions
entity right-click actions
custom AI
future robots/lights/science data

CUSTOM MODEL COMPATIBILITY TIP
------------------------------

Put "custom" before "template".
Use "template": "blocky_humanoid".
The entity still uses the custom mesh when customCubes is non-empty.

SUPPORTED ACTIONS
-----------------

message
say
giveBlock
spawnEntity
openMenu
closeMenu
placeBlock

FINAL CHECKLIST
---------------

[ ] pack_manifest.json is at the ZIP root.
[ ] JSON files are inside content/.
[ ] Optional copies are inside data/.
[ ] Block IDs are 0-255.
[ ] Blocks use color arrays like [255, 0, 0].
[ ] Commands start with /.
[ ] Entity type IDs use lowercase_with_underscores.
[ ] Custom entities put custom before template.
[ ] Menus have IDs.
[ ] Button actions use supported actions.
[ ] Right-click block IDs match registered block IDs.
[ ] The ZIP is placed in expansion_packs/.
[ ] Old duplicate versions are removed.
[ ] Console shows objects registering.

Manifest

Every pack needs a root pack_manifest.json.

{
  "packId": "my_first_pack",
  "name": "My First Pack",
  "version": "1.0.0",
  "author": "YourName",
  "description": "Adds test blocks, entities, commands, menus, and recipes.",
  "safeMode": true,
  "entryType": "data-pack",
  "entries": {
    "blocks": "content/blocks.json",
    "entities": "content/entities.json",
    "commands": "content/commands.json",
    "recipes": "content/recipes.json",
    "menus": "content/menus.json",
    "blockActions": "content/block_actions.json",
    "entityActions": "content/entity_actions.json",
    "customAI": "content/custom_ai.json",
    "robots": "content/robots.json",
    "lights": "content/lights.json",
    "science": "content/science.json"
  }
}

Blocks Guide

Blocks go in content/blocks.json.

{
  "blocks": [
    {
      "id": 140,
      "name": "Ruby Block",
      "solid": true,
      "color": [190, 30, 55],
      "emission": 0,
      "tags": ["gem", "decorative"]
    },
    {
      "id": 141,
      "name": "Neon Blue Block",
      "solid": true,
      "color": [40, 180, 255],
      "emission": 14,
      "tags": ["light", "neon"]
    }
  ]
}
Block IDs must be 0-255. Color must be an RGB array.

Commands Guide

Commands go in content/commands.json.

{
  "commands": [
    {
      "name": "/ruby",
      "usage": "/ruby",
      "description": "Gives Ruby Block.",
      "action": "giveBlock",
      "target": "Ruby Block"
    },
    {
      "name": "/testmenu",
      "usage": "/testmenu",
      "description": "Opens the test menu.",
      "action": "openMenu",
      "target": "test_menu"
    }
  ]
}

Publish + ui.store

Upload packs to modUI/packs/ and list them in modUI/ui.store.

{
  "title": "IRL Simulator Mod Store",
  "message": "Download expansion packs for IRL Simulator.",
  "packs": [
    {
      "name": "My First Pack",
      "description": "Adds test blocks and commands.",
      "url": "https://yourname.github.io/modUI/packs/my_first_pack.zip"
    }
  ]
}
Use plain URL strings. Do not paste HTML anchor code into ui.store.

Troubleshooting

Pack does not load

Check that pack_manifest.json is at the ZIP root.

Blocks show 0

Check content/blocks.json and the top-level blocks array.

Block skipped

Check id 0-255 and color [r,g,b].

customCubes=0

Put custom before template and use template blocky_humanoid.

Menu does not show

Check runModAction openMenu support and modMenuOverlay.draw in paintComponent.

Right click does nothing

Check block ID and engine right-click action hook.

Deep Troubleshooting

Pack skipped

pack_manifest.json is missing or not at ZIP root.

Blocks = 0

Check blocks.json path and top-level blocks array.

customCubes = 0

Put custom before template and use blocky_humanoid.

Menu does not show

Check openMenu support and modMenuOverlay.draw.

Command does nothing

Check chatRunModCommand routes into runModAction.

Right click does nothing

Check blockId/entityType and engine hooks.
