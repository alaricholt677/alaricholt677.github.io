What Camera.js does

The goal of this script is simple: pick a target element using a CSS selector, render just that element, and return the screenshot as an image string.

1. Copies the document

It uses the page HTML and places it inside a hidden iframe so the original page is not visually disturbed.

2. Isolates the target

It keeps your chosen id, class, or element visible while everything else is hidden for a clean capture.

3. Returns an image

The final result can be saved, previewed in an img tag, downloaded, or sent somewhere else.

How to set it up

Step 1: Add Camera.js to your page

Put this script tag near the bottom of your HTML, before your own page script: 

<script src="https://alaricholt677.github.io/PKGS/capturer/camera.js"></script>

Step 2: Add something you want to capture

<div id="my-card">
  <h2>Hello!</h2>
  <p>This element will become an image.</p>
</div>

Step 3: Capture it

const image = await screenshotTargets("#my-card");

document.querySelector("#preview").src = image;

Step 4: Show the image

<img id="preview" alt="Captured result" />
Important

If your actual function in camera.js is not named screenshotTargets, change the demo call in this file to your real function name.

Returned image const

The script returns an image data URL. This page stores it in: 

const image = await screenshotTargets("#capture-target");

After capture, the image can be placed into an <img> tag, downloaded, or used in your own app.

Basic API example

This is the simplest way to use the script: 

async function runCapture() {
  const image = await screenshotTargets("#target");

  console.log(image);

  document.querySelector("#output").src = image;
}

Multiple selectors

If your script supports arrays, you can capture multiple pieces together: 

const image = await screenshotTargets([
  "#header",
  ".profile-card",
  "footer"
]);

Notes and troubleshooting

  • Use normal CSS selectors like #id, .class, or section article.
  • If the button says the function is missing, make sure camera.js loaded correctly.
  • If the image is blank, check that your selected element exists and has visible size.
  • If external images do not show, they may need CORS support from the image host.
  • Test from a local server or GitHub Pages instead of opening the file directly as file://.
