Real settings

Follow these steps to add the local browser video generator into any HTML page. This uses the Spudzy Vid / RealLifeVideo script and runs fully in the browser.

  1. Add the generator script before your main app script. Put this near the bottom of your HTML, right before your own JavaScript code. 
    <script src="https://alaricholt677.github.io/PKGS/spudzy-vid/spudzy-vid-js.js"></script>
  2. Create a prompt input, a generate button, and a preview area. These are the basic elements your page needs. 
    <textarea id="prompt" placeholder="Describe your video..."></textarea>
<button id="generate">Generate</button>
<div id="preview"></div>
  3. Call RealLifeVideo.generate() when the user clicks Generate. The generator returns a blob URL/data object that can be shown in a normal video tag. 
    document.getElementById("generate").onclick = async () => {
  const prompt = document.getElementById("prompt").value.trim();
  const preview = document.getElementById("preview");

  if (!prompt) {
    alert("Type a prompt first.");
    return;
  }

  if (!window.RealLifeVideo) {
    alert("Generator script did not load.");
    return;
  }

  preview.textContent = "Generating...";

  const result = await RealLifeVideo.generate(prompt, {
    width: 960,
    height: 540,
    fps: 30,
    seconds: 6,
    bitrate: 4500000,
    returnFrames: false
  });

  preview.innerHTML = "";

  const video = document.createElement("video");
  video.controls = true;
  video.autoplay = true;
  video.loop = true;
  video.muted = true;
  video.playsInline = true;
  video.src = result.url;

  preview.appendChild(video);
};
  4. Use a data URL if you want the video to survive page refreshes. Blob URLs disappear when the page reloads. Convert the generated blob into a data URL before saving it. 
    function blobToDataURL(blob) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result);
    reader.onerror = reject;
    reader.readAsDataURL(blob);
  });
}
  5. Save each generated video as its own localStorage item. This keeps every video separate with its prompt, upload state, social data, and hidden JSON metadata. 
    const id = Date.now().toString(36);

const dataUrl = await blobToDataURL(result.blob);

const videoData = {
  id,
  prompt,
  createdAt: new Date().toISOString(),
  uploaded: false,
  dataUrl,
  uploadData: {
    likes: 0,
    dislikes: 0,
    followed: false,
    comments: []
  },
  social: {
    liked: false,
    disliked: false,
    followed: false
  },
  result: {
    ok: result.ok,
    mimeType: result.mimeType,
    width: result.width,
    height: result.height,
    frameCount: result.frameCount,
    correctedPrompt: result.correctedPrompt,
    parsed: result.parsed
  }
};

localStorage.setItem("my_video_" + id, JSON.stringify(videoData));
  6. Decide whether the video appears in Drafts or Feed. If uploaded is false, show it in Drafts. If uploaded is true, show it in the Feed. 
    if (videoData.uploaded) {
  // Show in feed
} else {
  // Show in drafts
}
  7. Upload a draft by changing its uploaded value. This moves it from Drafts into Feed. 
    videoData.uploaded = true;
videoData.uploadedAt = new Date().toISOString();

localStorage.setItem("my_video_" + videoData.id, JSON.stringify(videoData));
  8. Read saved videos back from localStorage. Loop through localStorage keys and find your saved video items. 
    const savedVideos = [];

for (let i = 0; i < localStorage.length; i++) {
  const key = localStorage.key(i);

  if (key.startsWith("my_video_")) {
    savedVideos.push(JSON.parse(localStorage.getItem(key)));
  }
}
  9. Render a saved video anywhere in your HTML. Data URLs can be used directly as the video source. 
    function renderSavedVideo(videoData, container) {
  const video = document.createElement("video");
  video.src = videoData.dataUrl;
  video.controls = true;
  video.loop = true;
  video.playsInline = true;

  container.appendChild(video);
}
  10. Keep storage limits in mind. Data URLs are persistent, but they are large. Shorter videos save more reliably. If localStorage quota is hit, export old videos or delete drafts.
